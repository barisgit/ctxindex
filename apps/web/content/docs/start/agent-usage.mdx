---
title: Use ctxindex from an agent
description: Give any shell-capable agent deterministic, realm-scoped context through the CLI.
---

There is no ctxindex MCP server or agent-specific SDK. Codex CLI, OpenClaw, Hermes-driven agents, Claude Code, and other code-executing agents use the same executable you do.

Give the agent this operating rule:

```text
Use ctxindex as the only interface to personal context. Start with `ctxindex
describe --format json` when you need the loaded vocabulary. Search with exact Realm
and Source filters where context matters. Treat returned ctx:// Refs as opaque,
pass them to get/thread/export/action, and read stable exit codes and warnings.
Never request secrets or send email; provider mutations stop at reversible Drafts.
```

The agent can then compose generic operations:

```sh
ctxindex search "Aurora kickoff" --realm work --format json
ctxindex get 'ctx://…' --format json
ctxindex thread 'ctx://…' --format json
ctxindex describe action mail.message.draft.create \
  --source work-mail \
  --format json
```

Use `--format json` for machine-readable command envelopes. Byte-stream exports use `--format json` when you want validated Profile JSON.

## Scope before retrieval

- Use `--realm company` when personal context must not be considered.
- Use `--source work-mail` when one collection must answer.
- Omit a Realm only when searching every eligible Realm is intentional.
- Inspect a typed Action before invoking it; a Source Adapter only implements declared Profile Actions.

For a longer reusable prompt and failure handling, see the [agent integration guide](/docs/guides/agent-integration).
