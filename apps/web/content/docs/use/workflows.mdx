title: Mail and calendar workflows
description: Apply the same generic commands to messages and Calendar Events across providers.
---

Source Adapters normalize provider records into portable Profiles. The CLI does not grow a Gmail, Outlook, or provider-specific command family.

## Mail

```sh
ctxindex search "invoice" \
  --realm company \
  --kind message \
  --format json
ctxindex get 'ctx://…/message/…' --format json
ctxindex thread 'ctx://…/message/…' --format json
```

Retrieve a complete parent message before creating a reply Draft. Then inspect the Source-specific Action schema and pass its complete typed input.

[Follow the mail workflow guide](/docs/guides/mail-workflows)

## Calendar

```sh
ctxindex search "design review" \
  --realm company \
  --kind calendar-event \
  --format json
ctxindex get 'ctx://…/calendar-event/…' --format json
```

Calendar Events use half-open intervals: `start` is inclusive and `end` is exclusive. All-day events use local dates; timed events use instants with timezone context.

[Follow the calendar workflow guide](/docs/guides/calendar-workflows)

## Across providers

One Realm can contain compatible Sources from multiple providers. Search merges normalized results deterministically while every Ref retains Source ownership for retrieval, Artifacts, and Actions.
