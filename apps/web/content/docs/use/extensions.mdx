---
title: Extensions and the local Marketplace
description: Install trusted Extensions directly or discover them through optional local Catalogs.
---

An Extension is trusted in-process TypeScript or JavaScript. It can be installed directly from npm, Git, or a local package; a Catalog is optional discovery and curation, not a required dependency graph.

## Install directly

```sh
ctxindex extension install npm @acme/ctxindex-mail@1.2.0 acme.mail --json
ctxindex extension install git \
  'git+https://github.com/acme/ctxindex-mail.git#4f3c2b1' \
  acme.mail \
  --json
ctxindex extension install local ./extensions/mail acme.mail --json
```

Package tooling resolves ordinary package dependencies. ctxindex selects one exact exported Extension root, validates the complete candidate registry, and publishes immutable managed bytes.

## Discover through a Catalog

A local Marketplace is the merged search projection of explicitly configured Git Catalog snapshots:

```sh
ctxindex extension catalog add community https://example.com/catalog.git \
  --ref refs/heads/main \
  --trust
ctxindex extension catalog search mail --json
ctxindex extension install catalog community acme.mail --json
```

Catalog add acknowledges repository acquisition. Installation is the separate executable boundary: it replays packages and imports trusted Extension code. Catalog search and Catalog-backed install refresh the involved Catalog by default; `--no-refresh` deliberately uses stored snapshots and reports their age.

Startup is offline: it reads installed records and managed bytes without refreshing Catalogs, contacting npm or Git, or invoking a package manager.

## Update and uninstall

```sh
ctxindex extension update acme.mail --json
ctxindex extension uninstall acme.mail --json
```

Updates explicitly re-pin installed code. Uninstall is blocked while configured Sources require the Extension unless `--force` preserves those Sources and data in an unavailable state.

[Open the generated Extension command reference](/docs/cli/extensions)

Installed Documentation Trees remain available offline through the CLI:

```sh
ctxindex docs list --json
ctxindex docs search "first sync" --json
ctxindex docs get acme.mail README.md
```
