# ctxindex V1.1 milestone

Status: completed on 2026-07-16. This is a historical scope record, not current
CLI or behavior documentation. Use [`openspec/specs/`](../../openspec/specs/)
and the installed CLI help for current truth.

## Outcome

An agent could use multiple Google and Microsoft Accounts across explicit
Realms: Gmail and Outlook mailboxes, Google and Microsoft calendars, local
files, and reversible Gmail/Outlook Drafts. ctxindex still could not send mail
or mutate calendars.

## Delivered scope

- Explicit Keychain/encrypted-file secret backends without silent fallback.
- Provider-neutral OAuth, stable Accounts and Grants, exact dynamic scope
  unions, refresh-token rotation, and bounded provider hosts.
- Strict `calendar.event@1` vocabulary with read-only Google and Microsoft
  calendar synchronization and retrieval.
- Microsoft mailbox search/retrieval, immutable ids, thread Relations,
  attachments, EML/JSON export, and Outlook Draft create/update.
- Multi-Account, multi-Realm inventory and compiled generic-command workflows.
- Mocked provider, egress, no-send, package, architecture, and human-assisted
  live acceptance gates.

## Deliberately deferred

- send scopes and send/reply-send/forward-send operations;
- calendar mutation, invitations, and RSVP Actions;
- Exchange on-premises, IMAP/SMTP, service accounts, and application
  permissions;
- webhooks/push subscriptions, cloud drives, contacts, and tasks;
- cross-source deduplication and provider-independent recurrence expansion.

Later work replaced the planned Client vocabulary with OAuth Apps and changed
several commands. The obsolete command transcript is intentionally omitted.
