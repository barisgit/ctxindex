# ctxindex V1.0 milestone

Status: completed. This is a historical scope record, not current CLI or
behavior documentation. Use [`openspec/specs/`](../../openspec/specs/) for
contracts and [`ctxindex --help`](../../README.md) for the installed interface.

## Outcome

An agent could use one deterministic CLI to discover, retrieve, and locally
materialize context across explicit Realms, then persist a reversible email
Draft through the same Source and authorization boundary.

## Delivered scope

- Public Profile, Source Adapter, and Extension factories with validated
  registries and trusted external TypeScript/JavaScript loading.
- Registry-derived kinds, fields, formats, Source configuration, Actions, and
  reference documentation.
- Explicit Realms and globally labeled Sources; no implicit or `global` Realm.
- Generic Resources, stable `ctx://` Refs, fields, chunks/FTS, Relations,
  Artifacts, sync bookkeeping, and purgeable materializations.
- One access model for sync, local/provider search, get, thread traversal,
  Artifact download, and Profile exports.
- Typed provider Actions limited to reversible email Draft creation/update.
- Gmail mailbox and local-directory verticals plus a compiled external-Extension
  proof on pinned Bun 1.3.14.

## Deliberately deferred

- sending email and non-email provider mutations;
- arbitrary Extension commands or workflow policy;
- out-of-process/non-TypeScript Adapters and Extension-private storage;
- multiple primary Profiles or cross-source identity merging;
- semantic search, push notifications, and compatibility with disposable
  pre-release data.

Later milestones changed names and command shapes. This file intentionally does
not preserve the obsolete planned CLI transcript.
